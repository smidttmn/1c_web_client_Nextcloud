# 🚀 ПОВТОРНАЯ УСТАНОВКА one_c_web_client_v3

**Версия:** 6.0.1 - Исправлен маршрут (redirect to dashboard fix)

---

## ✅ ЧТО ИСПРАВЛЕНО

**Проблема:** При открытии клиентской части перебрасывало на дашборд

**Причина:** В `appinfo/info.xml` маршрут использовал неправильный APP_ID

**Исправление:**
```xml
<!-- БЫЛО (не работает) -->
<route>one_c_web_client.page.index</route>

<!-- СТАЛО (работает) -->
<route>one_c_web_client_v3.page.index</route>
```

---

## 📋 ИНСТРУКЦИЯ ПО УСТАНОВКЕ

### Шаг 1: Удалите старую версию

```bash
# Подключитесь к серверу
ssh smidt@10.1.72.70
# Пароль: Apple0589

# Удалите старую версию
sudo -u www-data php /var/www/nextcloud/occ app:disable one_c_web_client_v3
sudo -u www-data php /var/www/nextcloud/occ app:remove one_c_web_client_v3
sudo rm -rf /var/www/nextcloud/apps/one_c_web_client_v3
sudo -u www-data php /var/www/nextcloud/occ maintenance:repair
```

### Шаг 2: Распакуйте новый пакет

```bash
cd /tmp
tar -xzf one_c_web_client_v3_package.tar.gz
cd install_package
```

### Шаг 3: Запустите установщик

```bash
chmod +x install.sh
sudo ./install.sh
```

### Шаг 4: Проверьте установку

```bash
# Проверка приложения
sudo -u www-data php /var/www/nextcloud/occ app:list | grep one_c

# Должно быть: one_c_web_client_v3: 6.0.1
```

## 🔧 ЕСЛИ ВСЁ ЕЩЁ ПЕРЕБРАСЫВАЕТ НА ДАШБОРД

### Проверьте маршруты

```bash
# Проверьте info.xml
cat /var/www/nextcloud/apps/one_c_web_client_v3/appinfo/info.xml | grep route

# Должно быть:
# <route>one_c_web_client_v3.page.index</route>
```

### Очистите кэш Nextcloud

```bash
sudo -u www-data php /var/www/nextcloud/occ maintenance:repair
sudo -u www-data php /var/www/nextcloud/occ memcache:clear 2>/dev/null || true
sudo systemctl restart apache2
```

### Проверьте логи

```bash
# Лог Nextcloud
sudo -u www-data php /var/www/nextcloud/occ log:read | tail -50

# Лог Apache
sudo tail -50 /var/log/apache2/error.log
```
