## 🔍 НАЙДЕННЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### 1. Неправильный маршрут в info.xml
**Проблема:** `<route>one_c_web_client.page.index</route>`  
**Решение:** `<route>one_c_web_client_v3.page.index</route>`

### 2. JavaScript открывает напрямую, а не через прокси
**Проблема:** `openDatabase(url, dbName)`  
**Решение:** Извлекать путь из URL и добавлять `/one_c_web_client_v3/`

### 3. 1С требует слэш на конце URL
**Проблема:** `/sgtbuh` → 404  
**Решение:** Добавлять слэш: `/sgtbuh/`

```javascript
const pathWithSlash = urlPath.endsWith('/') ? urlPath : urlPath + '/';
```

### 4. .htaccess блокирует ProxyPass
**Проблема:** `AllowOverride All` разрешает .htaccess переопределять ProxyPass  
**Решение:** `AllowOverride None`

### 5. JavaScript файлы 1С загружаются напрямую
**Проблема:** 1С генерирует HTML с путями `/ru/scripts/...`  
**Решение:** mod_substitute для переписывания URL

```apache
AddOutputFilterByType SUBSTITUTE text/html
Substitute "s|href=\"/|href=\"/one_c_web_client_v3/|in"
Substitute "s|src=\"/|src=\"/one_c_web_client_v3/|in"
```

### 6. ProxyPass не переписывает пути
**Проблема:** `/one_c_web_client_v3/sgtbuh/` → `https://10.72.1.5/one_c_web_client_v3/sgtbuh/` (404)  
**Решение:** `/one_c_web_client_v3/sgtbuh/` → `https://10.72.1.5/sgtbuh/`

```apache
ProxyPass /one_c_web_client_v3 https://10.72.1.5/ retry=0 timeout=60
```

### 7. Куки не работают
**Проблема:** Куки устанавливаются для 10.72.1.5, а не cloud.smidt.keenetic.pro  
**Решение:** ProxyPassReverseCookieDomain

```apache
ProxyPassReverseCookieDomain 10.72.1.5 cloud.smidt.keenetic.pro
```

### 8. JavaScript запросы от 1С идут напрямую
**Проблема:** 1С делает AJAX запросы на `/sgtbuh/ru/...`  
**Решение:** ProxyPass для путей 1С

```apache
ProxyPass /sgtbuh https://10.72.1.5/sgtbuh
ProxyPass /zupnew https://10.72.1.5/zupnew
```

### 9. mod_substitute не переписывает все URL
**Проблема:** Флаг `in` не работает для всех случаев  
**Решение:** Использовать флаг `gin` (глобально, без учёта регистра)

```apache
Substitute "s|href=\"/|href=\"/one_c_web_client_v3/|gin"
```

## ✅ РАБОЧАЯ КОНФИГУРАЦИЯ (ИТОГОВАЯ)

### Apache (nextcloud-le-ssl.conf)

```apache
<VirtualHost *:443>
  DocumentRoot /var/www/nextcloud/
  ServerName cloud.smidt.keenetic.pro
  
  # one_c_web_client_v3 - Прокси для 1С
  SSLProxyEngine on
  SSLProxyVerify none
  SSLProxyCheckPeerCN off
  SSLProxyCheckPeerName off
  
  # Прокси для one_c_web_client_v3 (ОБЯЗАТЕЛЬНО ДО ИСКЛЮЧЕНИЙ!)
  ProxyPass /one_c_web_client_v3 https://10.72.1.5/ retry=0 timeout=60
  ProxyPassReverse /one_c_web_client_v3 https://10.72.1.5/
  
  # Прокси для всех путей one_c_web_client_v3
  ProxyPassMatch ^/one_c_web_client_v3/(.*)$ https://10.72.1.5/$1
  
  # Прокси для конкретных баз 1С (запросы от JavaScript)
  ProxyPass /sgtbuh https://10.72.1.5/sgtbuh
  ProxyPass /zupnew https://10.72.1.5/zupnew
  
  # Переписывание куки
  ProxyPassReverseCookieDomain 10.72.1.5 cloud.smidt.keenetic.pro
  ProxyPassReverseCookiePath / /
  
  # Переписывание URL в HTML ответе от 1С
  AddOutputFilterByType SUBSTITUTE text/html
  Substitute "s|href=\"/|href=\"/one_c_web_client_v3/|gin"
  Substitute "s|src=\"/|src=\"/one_c_web_client_v3/|gin"
  
  # Разрешение фреймов и CSP
  Header unset X-Frame-Options
  Header always set Content-Security-Policy "frame-ancestors 'self'; frame-src *; connect-src *; script-src 'self' 'unsafe-inline' 'unsafe-eval' *; style-src 'self' 'unsafe-inline' *;"
  
  # Отключаем .htaccess чтобы не блокировал прокси
  <Directory /var/www/nextcloud/>
    Require all granted
    AllowOverride None
    Options FollowSymLinks MultiViews
  </Directory>
  
  SSLCertificateFile /etc/letsencrypt/live/cloud.smidt.keenetic.pro/fullchain.pem
  SSLCertificateKeyFile /etc/letsencrypt/live/cloud.smidt.keenetic.pro/privkey.pem
  Include /etc/letsencrypt/options-ssl-apache.conf
</VirtualHost>
```

### JavaScript (index.js)

```javascript
// Извлекаем путь из URL (например, /sgtbuh из https://10.72.1.5/sgtbuh)
const urlPath = new URL(url).pathname;

// Добавляем слэш на конце если нет
const pathWithSlash = urlPath.endsWith('/') ? urlPath : urlPath + '/';

// Открываем через прокси Nextcloud
const proxyPath = '/one_c_web_client_v3' + pathWithSlash;

openDatabase(proxyPath, dbName);
```
