## 🚀 Установка

### На тестовый сервер (cloud.smidt.keenetic.pro)

```bash
# Перейдите в директорию
cd /home/smidt/nc1c

# Запустите установщик от root
sudo ./install_interactive.sh
```

### На боевой сервер (drive.nppsgt.com)

```bash
# На локальном компьютере
cd /home/smidt/nc1c

# Скопируйте на боевой сервер
scp one_c_web_client_v3_full.tar.gz root@drive.nppsgt.com:/tmp/

# На боевом сервере
ssh root@drive.nppsgt.com
cd /tmp
tar -xzf one_c_web_client_v3_full.tar.gz
sudo ./install_interactive.sh
```

## 📋 Что спросит установщик

1. **Путь к Nextcloud** - определит автоматически (обычно `/var/www/nextcloud`)
2. **Конфиг Apache** - определит автоматически (обычно `/etc/apache2/sites-available/nextcloud.conf`)
3. **Доменное имя** - подставит из конфига (например, `cloud.smidt.keenetic.pro`)
4. **SSL сертификаты** - если Let's Encrypt, подставит автоматически
5. **1С сервер** - введите адрес (например, `https://10.72.1.5`)
6. **Базы 1С** - идентификаторы баз (например, `buh`, `zup`)
7. **Версия приложения** - выберите `2` (v3 - рекомендуется)

## ✅ Что делает установщик

### Автоопределение

- ✅ Находит Nextcloud (occ, apps директория)
- ✅ Находит конфиг Apache
- ✅ Определяет SSL сертификаты (Let's Encrypt или другие)
- ✅ Определяет доменное имя
- ✅ Проверяет модули Apache

### Безопасная установка

- ✅ Создает резервные копии конфига Apache
- ✅ Проверяет синтаксис перед применением
- ✅ Безопасно дополняет существующий конфиг (НЕ заменяет!)
- ✅ Сохраняет все SSL настройки
- ✅ Устанавливает правильные права (www-data)
- ✅ Логирует все действия

### Настройка Apache

Добавляет в конфиг:
- SSL Proxy Engine для безопасного соединения с 1С
- ProxyPass исключения для статических файлов Nextcloud
- ProxyPass для 1С сервера
- CSP заголовки для разрешения фреймов
- X-Frame-Options заголовки

### Установка приложения

- Копирует файлы в `apps/one_c_web_client_v3`
- Устанавливает через `occ app:install`
- Очищает кэш (`maintenance:repair`)
- Перезапускает Apache

## 🔍 Диагностика

### Если ошибка 500

```bash
# Проверьте логи Apache
tail -f /var/log/apache2/error.log

# Проверьте доступность 1С
curl -k https://10.72.1.5/buh

# Проверьте логи Nextcloud
tail -f /var/www/nextcloud/data/nextcloud.log
```

### Если CSP блокировка

```bash
# Проверьте CSP заголовки в конфиге Apache
grep -i "Content-Security-Policy" /etc/apache2/sites-available/nextcloud.conf
```

### Если Mixed Content

- Настройте HTTPS на 1С сервере
- Или разрешите mixed content в браузере
