## 🔧 ДОБАВЛЕНИЕ СЕРВЕРОВ ПОСЛЕ УСТАНОВКИ

Если пропустили добавление при установке:

### 1. Откройте конфиг Apache

```bash
sudo nano /etc/apache2/sites-available/nextcloud-le-ssl.conf
```

### 2. Добавьте ПОСЛЕ строки `DocumentRoot`:

```apache
    # ===================================================================
    # one_c_web_client_v3 - Прокси для 1С
    # ===================================================================

    SSLProxyEngine on
    SSLProxyVerify none
    SSLProxyCheckPeerCN off
    SSLProxyCheckPeerName off

    ProxyPass /one_c_web_client_v3 https://10.72.1.5/one_c_web_client_v3 retry=0 timeout=60
    ProxyPassReverse /one_c_web_client_v3 https://10.72.1.5/one_c_web_client_v3
    ProxyPassReverseCookiePath / /

    Header unset X-Frame-Options
    Header always set Content-Security-Policy "frame-ancestors 'self'; frame-src *;"

    # ===================================================================
    # END one_c_web_client_v3
    # ===================================================================
```

### 3. Проверьте и перезапустите

```bash
sudo apache2ctl configtest
sudo systemctl restart apache2
```

## 🐛 РЕШЕНИЕ ПРОБЛЕМ

### 1С не открывается (ошибка 499)

**Проблема:** Браузер открывает напрямую, а не через прокси

**Решение:**
1. Очистите кэш браузера: `Ctrl+Shift+R`
2. Проверьте консоль (F12):
   ```
   one_c_web_client_v3: Opening via proxy: /one_c_web_client_v3/sgtbuh
   ```
3. Проверьте Network (F12 → Network):
   ```
   GET /one_c_web_client_v3/sgtbuh 200 OK
   ```

### Ошибка 404

**Проблема:** ProxyPass стоит ПОСЛЕ исключений

**Решение:**
```bash
# Проверьте порядок
sudo grep -n 'ProxyPass' /etc/apache2/sites-available/nextcloud-le-ssl.conf

# Должно быть ДО всех ProxyPass с !
# Если после - переустановите через скрипт
```

### Ошибка 502

**Проблема:** Apache не запускается

**Решение:**
```bash
# Проверьте синтаксис
sudo apache2ctl configtest

# Восстановите резервную копию
sudo cp /tmp/one_c_backup_*/apache_config.backup /etc/apache2/sites-available/nextcloud-le-ssl.conf
sudo systemctl restart apache2
```
