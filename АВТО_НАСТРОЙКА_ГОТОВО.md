**Дата:** 16 марта 2026  
**Версия:** 10.0.0

## 🎯 Что было сделано

### Проблема:

При установке приложения `one_c_web_client_v3` **прокси не настраивался автоматически**, что приводило к ошибке 404 при открытии 1С.

**Причина:** ProxyPass добавлялся **ПОСЛЕ** всех исключений (`ProxyPass /core !`, `/apps !` и т.д.).

---

## ✅ Решение

### 1. Обновлён установщик (`install_package/install.sh`)

**Версия:** 10.0.0

**Новые возможности:**

- ✅ **Автоматическое определение типа конфигурации Apache:**
  - SSL (порт 443) - `nextcloud-le-ssl.conf`
  - NON_SSL (порт 80) - `nextcloud.conf`

- ✅ **Универсальная функция `configure_apache_universal()`:**
  - Работает с обоими типами конфигураций
  - Добавляет ProxyPass **ДО** всех исключений (ПРАВИЛЬНО!)
  - Добавляет ProxyPassMatch для путей
  - Добавляет mod_substitute для переписывания URL
  - Отключает AllowOverride для работы прокси
  - Создаёт резервную копию
  - Проверяет синтаксис
  - Перезапускает Apache

- ✅ **Не ломает существующие настройки:**
  - SSL сертификаты остаются нетронутыми
  - Резервная копия создаётся автоматически

### 2. Создан отдельный скрипт настройки прокси

**Файл:** `setup_apache_proxy_auto.sh`

**Назначение:** Быстрая настройка прокси для уже установленного приложения.

**Возможности:**

- ✅ Автоматическое определение конфигурации (SSL/NON_SSL)
- ✅ Получение сервера 1С из Nextcloud (через occ)
- ✅ Если не получилось - запрос вручную
- ✅ Настройка прокси ПРАВИЛЬНО
- ✅ Перезапуск Apache

### 3. Создана документация

**Файлы:**

- `АВТОМАТИЧЕСКАЯ_НАСТРОЙКА_ПРОКСИ.md` - подробная инструкция
- `БЫСТРЫЙ_СТАРТ_ПРОКСИ.md` - краткий быстрый старт
- `install_package/README.md` - обновлена документация установщика

## 🎯 Как это работает

### Правильный порядок директив Apache:

```apache
# 1. SSL Proxy Settings (только для SSL)
SSLProxyEngine on
SSLProxyVerify none
SSLProxyCheckPeerCN off
SSLProxyCheckPeerName off

# 2. ProxyPass для one_c_web_client_v3 (ОБЯЗАТЕЛЬНО ДО ИСКЛЮЧЕНИЙ!)
ProxyPass /one_c_web_client_v3 https://10.72.1.5/ retry=0 timeout=60
ProxyPassReverse /one_c_web_client_v3 https://10.72.1.5/

# 3. ProxyPassMatch для всех путей
ProxyPassMatch ^/one_c_web_client_v3/(.*)$ https://10.72.1.5/$1

# 4. Прокси для путей 1С
ProxyPass /sgtbuh https://10.72.1.5/sgtbuh
ProxyPassReverse /sgtbuh https://10.72.1.5/sgtbuh

# 5. ИСКЛЮЧЕНИЯ для статических файлов Nextcloud
ProxyPass /core !
ProxyPass /apps !
ProxyPass /dist !
# ... и т.д.

# 6. Переписывание куки
ProxyPassReverseCookieDomain 10.72.1.5 cloud.smidt.keenetic.pro
ProxyPassReverseCookiePath / /

# 7. mod_substitute для переписывания URL
AddOutputFilterByType SUBSTITUTE text/html
Substitute "s|href=\"/|href=\"/one_c_web_client_v3/|in"
Substitute "s|src=\"/|src=\"/one_c_web_client_v3/|in"

# 8. Разрешение фреймов и CSP
Header unset X-Frame-Options
Header always set Content-Security-Policy "frame-ancestors 'self'; ..."
```

### Ключевые моменты:

1. **ProxyPass ДО исключений** - критично!
2. **AllowOverride None** - отключаем для работы прокси
3. **mod_substitute** - переписывает URL в HTML ответе от 1С
4. **SSL Proxy Settings** - только для HTTPS прокси

## ✅ Проверка работы

### 1. Проверьте конфигурацию Apache:

```bash
sudo grep -A 30 "one_c_web_client_v3 - Прокси" /etc/apache2/sites-available/nextcloud-le-ssl.conf
```

Должно быть:
- `ProxyPass /one_c_web_client_v3` **ДО** `ProxyPass /core !`
- `SSLProxyEngine on` (для SSL)
- `mod_substitute` директивы

### 2. Проверьте синтаксис:

```bash
sudo apache2ctl configtest
```

Должно быть: `Syntax OK`

### 3. Обновите страницу в браузере:

```
Ctrl + Shift + R
```

### 4. Нажмите на кнопку базы 1С

**1С должна открыться!**
