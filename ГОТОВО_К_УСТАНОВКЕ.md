## 🔧 Что исправлено в версии 3.2.0

### Проблема в продакшн:

**Ошибка:** "Ошибка в конфигурации Apache"

**Причина:** Директивы `ProxyPass`, `SSLProxyEngine`, `Header` были добавлены **ПОСЛЕ** закрывающего тега `</VirtualHost>`, а должны быть **ВНУТРИ** блока `<VirtualHost *:443>`.

### Решение:

В скрипте `install_production.sh` директивы добавляются **ПЕРЕД** закрывающим тегом `</VirtualHost>`:

```bash
# Находим строку с </VirtualHost>
vhost_line=$(grep -n "</VirtualHost>" "$APACHE_CONFIG" | head -1 | cut -d: -f1)

# Вставляем директивы перед </VirtualHost>
line_before=$((vhost_line - 1))
sed -i "${line_before}r $directives_file" "$APACHE_CONFIG"
```

### Правильная структура конфига:

```apache
<VirtualHost *:443>
   ServerName localhost
   ServerAlias drive.technoorganic.info
   DocumentRoot /var/www/html/nextcloud

   # ... другие директивы ...

   # ===================================================================
   # one_c_web_client_v3 - Прокси для 1С (ВНУТРИ VirtualHost!)
   # ===================================================================

   SSLProxyEngine on
   SSLProxyVerify none
   SSLProxyCheckPeerCN off
   SSLProxyCheckPeerName off

   ProxyPass /core !
   ProxyPass /apps !
   ProxyPass /dist !
   ProxyPass /js !
   ProxyPass /css !
   ProxyPass /l10n !
   ProxyPass /index.php !

   ProxyPass /one_c_web_client_v3 https://10.72.1.5/one_c_web_client_v3
   ProxyPassReverse /one_c_web_client_v3 https://10.72.1.5/one_c_web_client_v3

   Header unset X-Frame-Options
   Header always set Content-Security-Policy "..."

</VirtualHost>  # <-- Закрывающий тег ПОСЛЕ всех директив

ErrorLog ${APACHE_LOG_DIR}/nextcloud/nextcloud_error.log
```
