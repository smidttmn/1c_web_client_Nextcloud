## 🐛 БЫВШАЯ ПРОБЛЕМА

**Симптомы:**
- Ошибка 500 при загрузке клиентской части
- Ошибка 500 при загрузке страницы настроек
- Невозможно зайти в настройки сервера

**Причина:**
В `Application.php` использовался `IServerContainer` с type hint, что вызывало ошибку в Nextcloud 33:

```php
// БЫЛО (не работает в NC 33)
$context->registerService(PageController::class, function(IServerContainer $c) {
    return new PageController(...);
});
```

**Ошибка:**
```
TypeError: Argument #1 ($c) must be of type OCP\IServerContainer,
OC\AppFramework\DependencyInjection\DIContainer given
```

---

## ✅ ИСПРАВЛЕНИЕ

**Обновлён файл `lib/AppInfo/Application.php`:**

```php
// СТАЛО (работает в NC 33)
$context->registerService(PageController::class, function($c) {
    return new PageController(
        $c->get('AppName'),
        $c->get(IRequest::class),
        $c->get(IConfig::class),
        $c->get(IL10N::class),
        $c->get(IURLGenerator::class)
    );
});
```

**Изменения:**
1. ✅ Удалён type hint `IServerContainer`
2. ✅ Удалены лишние импорты (`IServerContainer`, `SettingsManager`)
3. ✅ Удалена регистрация `AdminSettings`/`AdminSection` (не нужна для NC 33)
4. ✅ Используется `$c->get('AppName')` вместо `$this->getAppName()`
