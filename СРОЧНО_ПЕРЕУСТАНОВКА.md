# 🔧 СРОЧНО! Исправление ошибки с registerAdminSettings

**Проблема:** `Call to undefined method IRegistrationContext@anonymous::registerAdminSettings()`

**Причина:** В Nextcloud 30 **НЕТ** методов `registerAdminSettings()` и `registerAdminSection()` в `IRegistrationContext`

**Решение:** Обновлён архив с приложением (v3.2.3)

---

## 🚀 ПЕРЕУСТАНОВКА (5 минут)

### Шаг 1: Отключите старую версию

```bash
cd /var/www/html/nextcloud
sudo -u www-data php occ app:disable one_c_web_client_v3
```

### Шаг 2: Удалите приложение

```bash
sudo -u www-data php occ app:remove one_c_web_client_v3
```

### Шаг 3: Удалите файлы

```bash
rm -rf /var/www/html/nextcloud/apps/one_c_web_client_v3
```

### Шаг 4: Очистите кэш OPcache (ВАЖНО!)

```bash
# Перезапустите PHP-FPM
systemctl restart php8.1-fpm

# ИЛИ очистите OPcache вручную
echo "<?php opcache_reset(); ?>" | php
```

### Шаг 5: Распакуйте НОВУЮ версию

```bash
cd /var/www/html/nextcloud/apps
tar -xzf /tmp/one_c_web_client_v3_fixed.tar.gz
mv one_c_web_client_v3_clean one_c_web_client_v3
```

### Шаг 6: Установите права

```bash
chown -R www-data:www-data one_c_web_client_v3
chmod -R 755 one_c_web_client_v3
```

### Шаг 7: Установите приложение

```bash
cd /var/www/html/nextcloud
sudo -u www-data php occ app:install one_c_web_client_v3
```

### Шаг 8: Очистите кэш

```bash
sudo -u www-data php occ maintenance:repair
sudo -u www-data php occ memcache:clear 2>/dev/null || true
```

### Шаг 9: Перезапустите Apache

```bash
systemctl restart apache2
```

## 📋 ЧТО ИСПРАВЛЕНО

### Было (НЕ работает в NC 30):

```php
// ❌ Эти методы НЕ СУЩЕСТВУЮТ в NC 30
$context->registerAdminSettings(AdminSettings::class);
$context->registerAdminSection(AdminSection::class);
```

### Стало (работает в NC 30-32):

```php
// ✅ Регистрируем как сервисы
$context->registerService(AdminSettings::class, function(IServerContainer $c) {
    return new AdminSettings($c->get(IConfig::class));
});

$context->registerService(AdminSection::class, function(IServerContainer $c) {
    return new AdminSection(
        $c->get(IL10N::class),
        $c->get(IURLGenerator::class)
    );
});
```

## 🐛 ЕСЛИ ОШИБКА ОСТАЛАСЬ

### 1. Проверьте что файлы обновлены

```bash
# Проверьте версию Application.php
grep -A5 "public function register" /var/www/html/nextcloud/apps/one_c_web_client_v3/lib/AppInfo/Application.php
```

**Должно быть:**
```php
$context->registerService(AdminSettings::class, ...)
$context->registerService(AdminSection::class, ...)
```

**НЕ должно быть:**
```php
$context->registerAdminSettings(...)
$context->registerAdminSection(...)
```

### 2. Очистите кэш PHP OPcache

```bash
# Перезапустите PHP-FPM
systemctl restart php8.1-fpm

# ИЛИ удалите кэш OPcache
rm -rf /tmp/nextcloud-opcache-*
```

### 3. Проверьте что архив обновлён

```bash
# Проверьте дату архива
ls -lh /tmp/one_c_web_client_v3_fixed.tar.gz
```

**Должно быть:** свежая дата (сегодня)

### 4. Полная переустановка

```bash
# Удалите всё
sudo -u www-data php occ app:disable one_c_web_client_v3
sudo -u www-data php occ app:remove one_c_web_client_v3
rm -rf /var/www/html/nextcloud/apps/one_c_web_client_v3

# Очистите кэш
sudo -u www-data php occ maintenance:repair
sudo -u www-data php occ memcache:clear

# Перезапустите PHP
systemctl restart php8.1-fpm

# Установите заново
cd /var/www/html/nextcloud/apps
tar -xzf /tmp/one_c_web_client_v3_fixed.tar.gz
mv one_c_web_client_v3_clean one_c_web_client_v3
chown -R www-data:www-data one_c_web_client_v3
chmod -R 755 one_c_web_client_v3

sudo -u www-data php occ app:install one_c_web_client_v3
sudo -u www-data php occ maintenance:repair

# Перезапустите Apache
systemctl restart apache2
```
