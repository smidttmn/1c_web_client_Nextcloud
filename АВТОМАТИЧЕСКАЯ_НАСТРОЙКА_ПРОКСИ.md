## 📦 Вариант 1: Автоматическая настройка при установке приложения

### Запустите установщик:

```bash
sudo /path/to/install_package/install.sh
```

### Что делает установщик:

1. ✅ Находит Nextcloud (автоматически)
2. ✅ Находит конфигурацию Apache (SSL или без SSL)
3. ✅ Проверяет модули Apache (включает отсутствующие)

5. ✅ **Интерактивно запрашивает серверы 1С**
6. ✅ **Автоматически настраивает прокси:**
   - Определяет тип конфигурации (SSL порт 443 или NON_SSL порт 80)

   - Добавляет `ProxyPassMatch` для путей

8. ✅ Перезапускает Apache
9. ✅ Создаёт резервную копию конфигурации

## 🔧 Как это работает

### Правильная конфигурация ProxyPass:

```apache
# 1. SSL Proxy Settings (только для SSL)
SSLProxyEngine on
SSLProxyVerify none
SSLProxyCheckPeerCN off
SSLProxyCheckPeerName off

# 2. ProxyPass для one_c_web_client_v3 (ОБЯЗАТЕЛЬНО ДО ИСКЛЮЧЕНИЙ!)
ProxyPass /one_c_web_client_v3 https://10.72.1.5/ retry=0 timeout=60
ProxyPassReverse /one_c_web_client_v3 https://10.72.1.5/

# 3. ProxyPassMatch для всех путей
ProxyPassMatch ^/one_c_web_client_v3/(.*)$ https://10.72.1.5/$1

# 4. Прокси для путей 1С (sgtbuh, zupnew и т.д.)
ProxyPass /sgtbuh https://10.72.1.5/sgtbuh
ProxyPassReverse /sgtbuh https://10.72.1.5/sgtbuh

# 5. ИСКЛЮЧЕНИЯ для статических файлов Nextcloud
ProxyPass /core !
ProxyPass /apps !
ProxyPass /dist !
# ... и т.д.

# 6. Переписывание куки
ProxyPassReverseCookieDomain 10.72.1.5 cloud.smidt.keenetic.pro
ProxyPassReverseCookiePath / /

# 7. mod_substitute для переписывания URL
AddOutputFilterByType SUBSTITUTE text/html
Substitute "s|href=\"/|href=\"/one_c_web_client_v3/|in"
Substitute "s|src=\"/|src=\"/one_c_web_client_v3/|in"

# 8. Разрешение фреймов и CSP
Header unset X-Frame-Options
Header always set Content-Security-Policy "frame-ancestors 'self'; ..."
```

### Ключевые моменты:

1. **ProxyPass ДО исключений** - это критично! Если поставить после `ProxyPass /apps !`, прокси не будет работать.

2. **AllowOverride None** - отключаем, чтобы `.htaccess` не блокировал прокси.

3. **mod_substitute** - переписывает URL в HTML ответе от 1С.

4. **SSL Proxy Settings** - требуются только для HTTPS прокси.

## 🧪 Проверка работы

### 1. Проверьте конфигурацию Apache:

```bash
sudo grep -A 30 "one_c_web_client_v3 - Прокси" /etc/apache2/sites-available/nextcloud-le-ssl.conf
```

Должно быть:
- `ProxyPass /one_c_web_client_v3` **ДО** `ProxyPass /core !`
- `SSLProxyEngine on` (для SSL)
- `mod_substitute` директивы

### 2. Проверьте синтаксис:

```bash
sudo apache2ctl configtest
```

Должно быть: `Syntax OK`

### 3. Проверьте статус Apache:

```bash
sudo systemctl status apache2
```

### 4. Обновите страницу в браузере:

```
Ctrl + Shift + R
```

### 5. Нажмите на кнопку базы 1С

**1С должна открыться!**

## 🆘 Устранение проблем

### Ошибка 404 при открытии 1С:

```bash
# Проверьте, что ProxyPass добавлен ДО исключений
sudo grep -B 5 -A 5 "ProxyPass /one_c_web_client" /etc/apache2/sites-available/nextcloud-le-ssl.conf

# Если после исключений - запустите скрипт настройки
sudo ./setup_apache_proxy_auto.sh
```

### Ошибка 502 Bad Gateway:

```bash
# Проверьте, доступен ли сервер 1С
curl -k https://10.72.1.5/sgtbuh/

# Проверьте логи Apache
sudo tail -f /var/log/apache2/error.log
```

### Mixed Content (блокировка браузером):

- Откройте настройки браузера
- Разрешите небезопасный контент для вашего домена
- Или настройте HTTPS на сервере 1С

### 1С не загружается полностью:

```bash
# Проверьте mod_substitute
sudo a2enmod substitute
sudo systemctl restart apache2

# Проверьте логи браузера (F12 → Console)
```
