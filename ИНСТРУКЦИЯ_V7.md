## 📋 ПОШАГОВАЯ ИНСТРУКЦИЯ

### Шаг 1: Удалите старую версию

```bash
# Подключитесь к серверу
ssh smidt@10.1.72.70
# Пароль: Apple0589

# Удалите старую версию
sudo -u www-data php /var/www/nextcloud/occ app:disable one_c_web_client_v3
sudo -u www-data php /var/www/nextcloud/occ app:remove one_c_web_client_v3
sudo rm -rf /var/www/nextcloud/apps/one_c_web_client_v3
sudo -u www-data php /var/www/nextcloud/occ maintenance:repair
```

### Шаг 2: Распакуйте пакет v7.0.0

```bash
cd /tmp
tar -xzf one_c_web_client_v3_package.tar.gz
cd install_package
```

### Шаг 3: Запустите установщик

```bash
chmod +x install.sh
sudo ./install.sh
```

## 🔧 ДОБАВЛЕНИЕ СЕРВЕРОВ ПОСЛЕ УСТАНОВКИ

Если пропустили добавление при установке:

### 1. Откройте конфиг Apache

```bash
sudo nano /etc/apache2/sites-available/nextcloud-le-ssl.conf
```

### 2. Добавьте перед `</VirtualHost>`:

```apache
    # ===================================================================
    # one_c_web_client_v3 - Прокси для 1С
    # ===================================================================

    SSLProxyEngine on
    SSLProxyVerify none
    SSLProxyCheckPeerCN off
    SSLProxyCheckPeerName off

    ProxyPass /core !
    ProxyPass /apps !
    ProxyPass /dist !
    ProxyPass /js !
    ProxyPass /css !
    ProxyPass /l10n !
    ProxyPass /index.php !

    # Прокси для сервера 1: https://10.72.1.5
    ProxyPass /one_c_web_client_v3 https://10.72.1.5/one_c_web_client_v3
    ProxyPassReverse /one_c_web_client_v3 https://10.72.1.5/one_c_web_client_v3

    # Прокси для сервера 2: https://10.72.1.6
    ProxyPass /one_c_web_client_v3 https://10.72.1.6/one_c_web_client_v3
    ProxyPassReverse /one_c_web_client_v3 https://10.72.1.6/one_c_web_client_v3

    Header unset X-Frame-Options
    Header always set Content-Security-Policy "frame-ancestors 'self'; frame-src *;"

    # ===================================================================
    # END one_c_web_client_v3
    # ===================================================================
```

### 3. Проверьте и перезапустите

```bash
sudo apache2ctl configtest
sudo systemctl restart apache2
```
