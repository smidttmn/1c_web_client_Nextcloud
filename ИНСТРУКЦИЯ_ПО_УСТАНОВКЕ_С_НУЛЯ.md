## 📋 Шаг 2: Проверка перед установкой

### 2.1. Убедитесь, что Nextcloud работает:

```bash
# Проверка статуса Nextcloud
sudo -u www-data php /var/www/nextcloud/occ status

# Должно быть:
# - installed: true
# - maintenance: false
```

### 2.2. Проверьте Apache:

```bash
# Проверка статуса Apache
sudo systemctl status apache2

# Должно быть: active (running)
```

### 2.3. Проверьте модули Apache:

```bash
# Проверка необходимых модулей
sudo a2query -m proxy proxy_http headers rewrite substitute ssl

# Если какой-то модуль не включён - установщик его включит
```

## 📋 Шаг 7: Проверка через консоль (опционально)

### 7.1. Проверьте приложение:

```bash
sudo -u www-data php /var/www/nextcloud/occ app:list | grep one_c
```

**Должно быть:**
```
one_c_web_client_v3: 10.0.0
```

### 7.2. Проверьте конфигурацию Apache:

```bash
sudo grep -A 5 "ProxyPass /one_c_web_client_v3" /etc/apache2/sites-available/nextcloud-le-ssl.conf
```

**Должно быть:**
```apache
ProxyPass /one_c_web_client_v3 https://10.72.1.5/ retry=0 timeout=60
ProxyPassReverse /one_c_web_client_v3 https://10.72.1.5/
```

### 7.3. Проверьте, что ProxyPass ДО исключений:

```bash
sudo grep -B 10 "ProxyPass /core !" /etc/apache2/sites-available/nextcloud-le-ssl.conf | tail -15
```

**Должно быть:**
```apache
ProxyPass /one_c_web_client_v3 https://10.72.1.5/ ...
ProxyPassMatch ^/one_c_web_client_v3/(.*)$ https://10.72.1.5/$1

# 4. ИСКЛЮЧЕНИЯ для статических файлов Nextcloud
ProxyPass /core !
ProxyPass /apps !
```

**ProxyPass должен быть ДО `ProxyPass /core !`**

## 🆘 Устранение проблем

### Ошибка 404 при открытии 1С:

**Проблема:** ProxyPass добавлен после исключений

**Решение:**
```bash
sudo ./setup_apache_proxy_auto.sh
```

### Ошибка 502 Bad Gateway:

**Проблема:** Сервер 1С недоступен

**Решение:**
```bash
# Проверьте доступность сервера 1С
curl -k https://10.72.1.5/sgtbuh/

# Проверьте логи Apache
sudo tail -f /var/log/apache2/error.log
```

### Mixed Content (блокировка браузером):

**Проблема:** Браузер блокирует HTTP ресурсы на HTTPS странице

**Решение:**
1. Откройте настройки браузера
2. Найдите "Небезопасный контент" или "Mixed Content"
3. Разрешите для вашего домена
4. ИЛИ настройте HTTPS на сервере 1С

### 1С не загружается полностью:

**Проблема:** mod_substitute не включён

**Решение:**
```bash
sudo a2enmod substitute
sudo systemctl restart apache2
```
